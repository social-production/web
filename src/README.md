# Frontend Scaffold

This is the initial frontend folder scaffold.

It is intentionally lightweight because the repo is still in the planning-to-build transition.

Use it like this:

- `routes/` owns route-level pages and layout files
- `lib/features/` owns product-domain composition
- `lib/components/cards/public-feed/` owns Public feed card files
- `lib/components/cards/personal-feed/` owns Personal feed card files
- `lib/components/cards/shared/` owns low-level reusable card primitives only
- `lib/services/adapters/` owns backend adapters
